# Relational Model

A **Relational Model** defines how data from one or more Relational Tables is **organized and connected**.

It serves as the structural layer between raw data and user-facing outputs, bringing together related entities into a meaningful and usable form.

## Purpose

Relational Models are used to:

- combine data from multiple tables
- define relationships between entities
- organize data for reporting, planning, and application use
- provide a structured foundation for Relational Views

## Key Concepts

- A model represents a **cohesive data structure** built around a specific business purpose
- Models connect tables in a way that reflects real-world relationships
- Models are reusable and can support multiple downstream views

## Typical Use Cases

- Workforce models combining employees and departments  
- Asset models linking assets to locations  
- Product and service models for planning and analysis  

## Notes

- A well-designed model simplifies downstream configuration
- Models should be focused on a clear business use case
- Models act as the bridge between tables and views
