# Relationships

[← Back to Relational Models Overview](index.md)

The **Relationships** page is used to define how sources within a relational model are connected.

These relationships allow the model to combine multiple entities into a cohesive and usable structure.

## Overview

Use the Relationships page to:

- define links between model sources
- create meaningful cross-table structures
- support downstream data access and display
- organize how related data is brought together in the model

## Typical Use Cases

Examples include:

- linking employees to departments
- linking assets to locations
- linking products to categories or service lines

## Design Guidance

When defining model relationships:

- reflect real-world business structure
- keep joins and links easy to understand
- avoid unnecessary complexity
- make sure each relationship supports a valid business use case

## Notes

- Relationships are central to the usefulness of a relational model.
- Strong relationship design improves downstream view creation and user experience.
- Model relationships should build on well-defined source structures.
