# Tables

[← Back to Relational Tables Overview](index.md)

The **Tables** page is used to define and manage the core relational tables in Revfore Accelerate.

Relational tables represent the primary business entities in your solution and provide the foundation for the broader relational framework.

## Overview

Use the Tables page to:

- create new relational tables
- review existing table definitions
- manage table-level settings
- organize the core entities used throughout the solution

Each relational table should represent a clear business concept, such as a product, employee, asset, service, or location.

## Typical Use Cases

Use relational tables to define entities such as:

- Employees
- Assets
- Products
- Services
- Locations

## Notes

- Tables should be designed around business entities rather than technical groupings.
- Keep table names clear and consistent.
- Tables are the starting point for columns, relationships, models, views, and forms.
